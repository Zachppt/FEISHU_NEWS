# 金融情报系统 · 心跳任务

每次心跳按以下流程执行，**优先跳过无效工作**：

## 执行流程

### 第一步：抓取新闻
运行 fetch-news skill。

**如果返回新增条数为 0**：
→ 直接输出 `HEARTBEAT_OK`，结束，不执行后续步骤。

**如果有新增条目**：
→ 继续第二步。

### 第二步：过滤新闻
运行 filter-news skill。

**如果过滤后命中条数为 0**：
→ 直接输出 `HEARTBEAT_OK`，结束，不执行 monitor。

**如果有命中条目**：
→ 继续第三步。

### 第三步：预警扫描
运行 monitor skill。

## 规则

- 没有新内容时直接输出 `HEARTBEAT_OK`，不推送任何频道
- 整个心跳执行时间控制在 **60 秒以内**
- 不主动推送板块汇总和情绪快照（由 Cron 负责）

## 静默时间

00:00–06:00 跳过 fetch-news 和 filter-news，只运行 monitor 扫描已有缓存中尚未推送的条目。
