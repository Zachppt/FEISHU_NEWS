# Filter News Skill

## 触发方式
- fetch-news 完成后自动触发
- 用户发送"过滤新闻"手动触发

## 执行步骤

### 1. 读取数据
```bash
cat ~/.openclaw/workspace/raw-news.json
cat ~/.openclaw/workspace/watchlist.json
cat ~/.openclaw/workspace/filtered-news.json
```

---

### 2. 第一层：硬性丢弃（格式/质量问题，直接淘汰）

以下情况无论内容如何，一律丢弃：

**格式问题：**
- 标题字数 < 10 字（中文）或 < 20 字符（英文）
- 标题全为大写字母（标题党特征）
- 标题含超过 3 个连续感叹号或问号
- 标题以"【广告】""【推广】""【赞助】"等标记开头
- 摘要字段为空且标题无法独立传达信息（判断依据：标题不含主语或事件）

**明确垃圾内容：**
- 标题或摘要含以下任意词：
  - 营销类：广告、推广、赞助、软文、植入、合作推广、品牌专区
  - 招聘类：招聘、求职、内推、社招、校招、JD、岗位职责
  - 促销类：优惠券、折扣、秒杀、限时、特惠、满减、红包、拼团、砍价
  - 博彩类：博彩、赌博、彩票、娱乐城、老虎机、百家乐
  - 诱导类：加微信、加我、私信我、点击领取、免费领

**重复内容：**
- URL 已存在于 filtered-news.json 中
- 标题与 filtered-news.json 中任意条目的标题**相似度超过 85%**（去掉标点和数字后逐字比对，匹配字符占比）
  - 目的：过滤"换皮转载"，保留同一事件的不同角度报道

---

### 3. 第二层：时效性过滤

金融新闻的时效性决定信息价值：

**基准时间**：取当前批次 raw-news.json 中**最新一条**的发布时间作为基准（不用系统当前时间，防止深夜新闻稀少时全部被丢弃）。

**分类处理：**

| 新闻类型 | 最大时效 | 判断依据 |
|---------|---------|---------|
| 突发预警（暴雷/退市/停牌/紧急政策） | 12 小时 | 含关键词：暴雷、退市、停牌、紧急、突发、重大 |
| 常规财经新闻 | 6 小时 | 默认 |
| 宏观数据发布（CPI/非农/GDP等） | 24 小时 | 含关键词：CPI、PPI、GDP、非农、PMI、零售销售 |
| 研报/分析文章 | 48 小时 | 来源含：研报、分析师、评级、目标价、维持 |
| 公司公告/监管文件 | 72 小时 | 来源含：交易所、证监会、公告、披露 |

---

### 4. 第三层：内容相关性过滤

通过以上两层后，必须命中以下任意一个维度才保留：

#### 维度A：个股相关
标题或摘要含 `watchlist.stocks` 中任意个股名称。

优先级提升（保留且标记为高优先级）：
- 含以下事件词：**业绩预告、利润预增/预减、分红、回购、增持、减持、并购、重组、分拆、定增、配股、停复牌、问询函、立案调查、实控人变更**
- 含财务数据：净利润、营收、毛利率、同比、环比 + 具体数字

#### 维度B：板块/行业相关
标题或摘要含 `watchlist.sectors` 中任意板块名称。

仅保留具有**板块共性意义**的新闻，丢弃以下情况：
- 仅提及板块名称但内容与该板块无实质关联（如"新能源概念股午后拉升"无具体驱动因素）
- 单纯的板块涨跌播报（无基本面信息）：如"半导体板块跌 2%，个股普跌" → 丢弃
- 含具体政策、技术突破、供需变化、龙头公司重大动态 → 保留

#### 维度C：宏观关键词
标题或摘要含 `watchlist.keywords` 中任意词，**或**含以下固定宏观词：

**货币政策：** 美联储、Fed、FOMC、降息、加息、利率决议、QE、缩表、点阵图、鲍威尔、央行、MLF、LPR、逆回购、存款准备金率

**经济数据：** GDP、CPI、PPI、PCE、非农就业、失业率、PMI、ISM、零售销售、工业产出、贸易差额、经常账户

**市场结构：** 北向资金、南向资金、融资融券、大宗交易、股权质押、ETF申赎

**信用风险：** 债券违约、评级下调、展期、流动性危机、信用利差、CDS

**监管政策：** 证监会、银保监、发改委、工信部、政策出台、征求意见稿、处罚、立案

#### 维度D：高价值事件（无论是否在 watchlist，只要命中就保留）

以下属于市场共识的重大事件，强制保留：
- **A股/港股/美股重大指数涨跌超 ±2%** 且有明确原因分析
- **上市公司 ST、*ST、退市风险警示**
- **实控人/大股东被调查、拘留、涉刑**
- **IPO 撤单、上市申请被否**
- **系统性风险信号**：银行挤兑、汇率异常波动、国债收益率急涨
- **地缘政治直接冲击市场**：战争、制裁、封锁影响到具体行业

---

### 5. 第四层：噪音精滤（通过前三层后的最后检查）

以下情况即使通过了前面的过滤，也降级或丢弃：

**降级丢弃（信息密度极低）：**
- 纯行情播报，无分析、无原因：如"茅台今日涨 1.2%，收报 1680 元" → 丢弃
- 重复的"分析师称"类文章，无新增信息，仅引用已知观点
- 预测类文章，无数据支撑，全文仅为"预计""或将""可能" → 丢弃
- 标题与摘要信息完全重复，无增量内容

**保留但标记低优先级：**
- 行业趋势类长文（信息密度中等，适合快报汇总，不适合即时预警）
- 海外市场收盘播报（有数据，但非实时）

---

### 6. 写入结果

将通过过滤的条目追加到 filtered-news.json（保留最新 500 条，原子写入）：
```bash
echo '<updated_filtered_json>' > ~/.openclaw/workspace/filtered-news.json.tmp && \
mv ~/.openclaw/workspace/filtered-news.json.tmp ~/.openclaw/workspace/filtered-news.json
```

每条记录新增以下字段：
- `matched_terms`：命中的关键词列表（便于 monitor/sentiment 快速定位）
- `match_dimension`：命中维度（stocks / sectors / macro / keywords / high_value）
- `priority`：优先级（high / normal / low）
- `filtered_at`：过滤时间戳（ISO8601）

---

### 7. 完成
静默结束，不输出任何内容。

---

## 注意事项
- **全程不调用任何 AI 或 LLM**，所有判断为确定性规则
- 去重用 `.hash // .id` 作为唯一键
- 标题相似度比对：去除标点符号、数字、空格后，计算公共字符占较短标题的比例，超过 85% 视为重复
- `priority: high` 的条目在 monitor 中优先检查
- 来源可信度不作为过滤依据（无法在 RSS 层面可靠判断），但可在 sentiment/summarize 中加权
